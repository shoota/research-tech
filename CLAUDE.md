# CLAUDE.md

## プロジェクト概要

Web技術の調査・ドキュメントサイト。Docusaurus 3 + GitHub Pages で構成。

## コマンド

- `npm start` - ローカル開発サーバー起動
- `npm run build` - 本番ビルド
- `npm run serve` - ビルド結果のローカル確認
- `npm run typecheck` - TypeScript 型チェック

## ディレクトリ構成

- `docs/` - ドキュメント (Markdown/MDX)
- `src/pages/` - 独立ページ (React)
- `src/css/` - カスタムCSS
- `static/` - 静的ファイル
- `docusaurus.config.ts` - サイト設定
- `sidebars.ts` - サイドバー定義

## ドキュメント作成規約

- `docs/` 配下に Markdown ファイルを追加
- Front matter に `id`, `title`, `sidebar_position`, `tags` を記載
- Front matter の `last_update.date` に調査実施日を記載（YYYY-MM-DD形式）
- 言語は日本語
- サイドバーはディレクトリ構造から自動生成（`sidebars.ts` で `autogenerated`）

### ディレクトリ構成規約

- `docs/<category>/` で技術カテゴリごとに分類
- カテゴリディレクトリに `_category_.json` を配置（`label`, `position`）
- ドキュメントID は `<category>-<topic>` 形式（例: `react-hooks`）

### ドキュメント品質基準

調査ドキュメントには以下のセクションを含める:

- **概要**: 何を調査したか（1-2文）
- **背景・動機**: なぜ調査したか
- **調査内容**: 技術の詳細、仕組み、特徴
- **検証結果**: 実際に試した結果（コード例付き）
- **まとめ**: 所感・プロジェクトへの適用可否
- **参考リンク**: 公式ドキュメント・記事のURL（番号付きリスト）

### 出典の記載規約

- 調査内容の記述で特定のソースに基づく情報には、文中に出典番号リンクを付ける
- 形式: `テキスト[[1]](#参考リンク)` のように参考リンクセクションへのアンカーリンクを使用する
- 参考リンクセクションは番号付きリスト（`1. [タイトル](URL)` 形式）で記載する
- **脚注記法（`[^1]` / `[^1]:`）は使用禁止**: Docusaurusが自動的に「Footnotes」セクションを生成し、参考リンクと重複するため
- 一般的に広く知られている事実や、複数ソースから確認できる内容には注釈不要
- コード例が特定の公式ドキュメントから引用・参考にしたものである場合も注釈を付ける

### コード例の規約

- コードブロックに言語指定を必須（```tsx, ```css 等）
- 実行可能なコード例を優先
- コード例にはコメントで説明を付ける
- ファイル名がある場合は `title` プロパティで明示（例: ```tsx title="App.tsx"）

## カスタムコマンド

- `/new-doc <category> <title>` - ドキュメント雛形を生成
- `/research <テーマ>` - Web調査→ドキュメント化ワークフロー
