ドキュメントの雛形を生成します。

引数: $ARGUMENTS（形式: `<category> <title>`、例: `react Hooks入門`）

## 手順

1. 引数を解析してカテゴリとタイトルを取得する
2. `docs/<category>/` ディレクトリが存在しなければ作成し、`_category_.json` も作成する
   - `_category_.json` の `label` はカテゴリ名を適切な日本語または英語で設定
   - `position` は既存カテゴリ数 + 1 で設定
3. ドキュメントIDは `<category>-<topic>` 形式で命名する（topicはタイトルから英語スラッグを生成）
4. CLAUDE.md のドキュメント品質基準に従い、以下のテンプレートで Markdown ファイルを生成する:

```markdown
---
id: <category>-<topic>
title: <title>
sidebar_position: <次の番号>
---

# <title>

## 概要

<!-- 何を調査したか（1-2文） -->

## 背景・動機

<!-- なぜ調査したか -->

## 調査内容

<!-- 技術の詳細、仕組み、特徴 -->

## 検証結果

<!-- 実際に試した結果（コード例付き） -->

## まとめ

<!-- 所感・プロジェクトへの適用可否 -->

## 参考リンク

- [公式ドキュメント](URL)
```

5. ファイル作成後、`npm run build` でビルドが通ることを確認する
6. 作成したファイルのパスを報告し、内容の執筆を促す
