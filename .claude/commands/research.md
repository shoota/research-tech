指定されたテーマについてWeb調査を行い、調査結果をドキュメントとしてまとめます。

引数: $ARGUMENTS（形式: `<category> <テーマ>`、例: `react React Server Components`）

## 手順

1. 引数を解析してカテゴリと調査テーマを取得する

2. 調査フェーズ:
   - WebSearch で最新の情報を検索する
   - context7 MCP（resolve-library-id → query-docs）で公式ドキュメントを取得する
   - WebFetch で重要なページの詳細を取得する
   - 複数ソースから情報を収集し、正確性を確認する

3. ドキュメント生成フェーズ:
   - `/new-doc` と同じ手順でカテゴリディレクトリとファイルを作成する
   - front matter の `last_update.date` に調査実施日（今日の日付）を YYYY-MM-DD 形式で記載する
   - CLAUDE.md のドキュメント品質基準に従い、各セクションを記述する:
     - **概要**: 調査テーマの簡潔な説明
     - **背景・動機**: このテーマが重要な理由
     - **調査内容**: 技術の詳細を具体的に記述
     - **検証結果**: コード例を含める（言語指定必須、コメント付き）
     - **まとめ**: 実用性の評価と所感
     - **参考リンク**: 調査で参照したURL一覧

4. 関連ドキュメント整理フェーズ:
   - `docs/` 配下の既存ドキュメントの front matter（title, id）を一覧取得し、今回作成したドキュメントと関連性があるものを特定する
   - 関連が見つかった場合:
     - 新規ドキュメントの概要セクション直後に `:::info 関連ドキュメント` ブロックで関連ドキュメントへのリンクを追加する（形式: `[タイトル](/docs/<category>/<id>)`）
     - 既存の関連ドキュメントにも新規ドキュメントへのリンクを追加する（既にリンクブロックがあれば追記、なければ新規作成）
   - 内容の重複が見つかった場合:
     - 重複箇所を特定し、一方に集約する（詳細はより適切なカテゴリのドキュメントに残す）
     - 集約元からはリンクで誘導する
   - 同一カテゴリ内に複数ドキュメントがある場合は、カテゴリ内の体系的な構成を確認し、必要に応じて sidebar_position を調整する

5. 検証フェーズ:
   - `npm run build` でビルドが通ることを確認する
   - エラーがあれば修正する

6. 完了報告:
   - 作成したファイルのパス
   - ドキュメントの概要サマリ
   - 関連ドキュメントとの紐付け結果（追加・更新したリンクの一覧）
   - `npm start` でプレビュー確認を促す
