## 概要

Claude Code のインタラクティブセッション中に `/` で呼び出せる組込みスラッシュコマンドの全一覧と、各コマンドの実行内容・効果・具体的なユースケースをまとめる。`/config` で設定可能な項目についても詳細に解説する。

:::info 関連ドキュメント
- [Claude Design — Anthropic Labs が提供するAIネイティブなデザイン環境](/docs/ai-tools/claude-code/claude-code-claude-design) — デザイン → 実装の handoff 元となる姉妹プロダクト
:::

## 背景・動機

Claude Code は CLI ベースの AI コーディングエージェントであり、セッション中に `/` プレフィックスで各種コマンドを呼び出せる。コマンドには大きく2種類あり、CLI に直接組み込まれた**ビルトインコマンド**と、プロンプトとして Claude に渡される**バンドルスキル**がある。コマンドの種類は多岐にわたるが、公式ドキュメントでは一覧表の形式で簡潔に記載されており、実際のユースケースまで踏み込んだ解説が不足している。本ドキュメントでは各コマンドを機能カテゴリごとに整理し、「いつ・どう使うか」を含めて網羅的にまとめる。

:::info 補足
コマンドの一部はプラットフォーム・プラン・環境に依存し、すべてのユーザーに表示されるわけではない。`/help` を実行すると、現在のセッションで利用可能なコマンドを確認できる[[1]](#参考リンク)。
:::

## 調査内容

### コマンドの分類

スラッシュコマンドは以下のカテゴリに大別できる。

| カテゴリ | 概要 |
|---------|------|
| セッション管理 | 会話の開始・終了・分岐・再開・リネーム |
| コンテキスト管理 | コンテキストの圧縮・可視化・巻き戻し |
| モデル・モード制御 | モデル切替・思考レベル・高速モード・プランモード |
| 情報・ステータス | 使用量・コスト・診断・統計 |
| 設定・権限 | 設定UI・権限管理・サンドボックス・テーマ |
| 統合・拡張 | GitHub連携・MCP・プラグイン・IDE・リモート |
| ワークフロー（バンドルスキル） | コードレビュー・一括変更・デバッグ・ループ・API開発 |
| その他 | ステッカー注文・パス共有・モバイル案内 |

### セッション管理

#### `/clear`

- **エイリアス**: `/reset`, `/new`
- **実行内容**: 会話履歴をクリアし、コンテキストを解放する。CLAUDE.md は保持される。
- **効果**: コンテキストウィンドウが完全にリセットされ、新しいセッションとして再スタートする。
- **ユースケース**:
  - 長時間の作業でコンテキストが肥大化し、応答精度が低下してきたとき
  - 別のタスクに切り替える際にクリーンな状態から始めたいとき
  - セッションの途中で方向性を完全にリセットしたいとき

#### `/compact [instructions]`

- **実行内容**: 会話履歴を要約してコンテキストを圧縮する。オプションで保持したい内容を指示できる。
- **効果**: コンテキストウィンドウの消費を抑えつつ、重要な情報は保持される。
- **ユースケース**:
  - コンテキストが上限に近づいてきたが、会話を継続したいとき
  - `/compact テスト結果と修正方針を保持` のように重要情報を指定して圧縮したいとき
  - 長いデバッグセッションで不要な試行錯誤の履歴を整理したいとき

#### `/resume [session]`

- **エイリアス**: `/continue`
- **実行内容**: 過去のセッションをID・名前で再開する。引数なしでセッションピッカーが開く。
- **効果**: 以前のコンテキストと会話履歴がそのまま復元される。
- **ユースケース**:
  - 前日に中断した作業を翌日続けたいとき
  - 複数のタスクを並行して進めており、特定のセッションに戻りたいとき
  - `claude -n "auth-refactor"` で名前をつけたセッションを `claude --resume auth-refactor` で再開する運用

#### `/branch [name]`

- **エイリアス**: `/fork`
- **実行内容**: 現在の会話をその時点で分岐し、新しいセッションを作成する。元のセッションは `/resume` で戻れる。
- **効果**: 実験的なアプローチを試す際に、元の会話を安全に保持したまま分岐できる。
- **ユースケース**:
  - ある修正方針を試したいが、うまくいかなければ元に戻したいとき
  - 同じ問題に対して複数の解決策を並行して検討したいとき
  - レビュー前に「この方向で進めてよいか」を確認する分岐点を作りたいとき

#### `/rename [name]`

- **実行内容**: 現在のセッションに名前を付ける（プロンプトバーにも表示される）。引数なしで自動生成。
- **効果**: セッション一覧で識別しやすくなり、`/resume` での再開が簡単になる。
- **ユースケース**:
  - 長期的なタスクに取り組む際に、後から見つけやすい名前を付けておきたいとき
  - `/rename auth-migration` のようにタスク名で管理するワークフロー

#### `/exit`

- **エイリアス**: `/quit`
- **実行内容**: Claude Code セッションを終了する。
- **ユースケース**: セッションを明示的に終了したいとき（`Ctrl+D` でも同様）

#### `/export [filename]`

- **実行内容**: 現在の会話をプレーンテキストとしてエクスポートする。ファイル名指定でファイルに書き出し、なしでクリップボードコピーかファイル保存のダイアログが開く。
- **ユースケース**:
  - 調査結果やデバッグの経過を共有資料としてまとめたいとき
  - チームメンバーに会話の内容を共有したいとき

#### `/copy [N]`

- **実行内容**: 最後のアシスタント応答をクリップボードにコピーする。`N` を指定するとN番目に古い応答をコピー。コードブロックがある場合はインタラクティブピッカーで選択可能。`w` キーでファイルへの書き出しも可能。
- **ユースケース**:
  - Claude が生成したコードスニペットだけを取り出してコピーしたいとき
  - SSH経由でリモートマシンから作業しており、クリップボード共有が使えないとき（`w` でファイルに書き出し）

### コンテキスト管理

#### `/context`

- **実行内容**: コンテキストウィンドウの使用状況をカラーグリッドで可視化する。最適化のための提案も表示される。
- **効果**: メモリ膨張やキャパシティ警告など、コンテキストの健全性を把握できる。
- **ユースケース**:
  - レスポンスが遅くなってきたとき、コンテキスト上限が原因か確認したいとき
  - `/compact` するタイミングの判断材料として定期的にチェック

#### `/rewind`

- **エイリアス**: `/checkpoint`
- **実行内容**: 会話やコード変更を以前の時点に巻き戻す。あるいは選択したメッセージ以降を要約する。
- **効果**: Claude が行ったファイル変更も含めて以前の状態に復元できる。
- **ユースケース**:
  - Claude が行った変更が期待と異なるとき、変更前の状態に戻したいとき
  - 「3つ前のターンまで戻してやり直したい」ケース
  - `Esc` + `Esc` のキーボードショートカットでも呼び出し可能

#### `/add-dir <path>`

- **実行内容**: 現在のセッションに追加の作業ディレクトリを加える。
- **効果**: 指定ディレクトリ内のファイルへの読み書きアクセスが可能になる。
- **ユースケース**:
  - モノレポで別パッケージのコードを参照しながら作業したいとき
  - 共通ライブラリのディレクトリを追加で参照対象にしたいとき

#### `/diff`

- **実行内容**: コミットされていない変更と、Claude のターンごとの差分をインタラクティブなビューアで表示する。左右キーでgit diff とターン別diffを切り替え、上下キーでファイルをブラウズ。
- **ユースケース**:
  - Claude が行った変更の全体像を確認してからコミットしたいとき
  - どのターンでどの変更が入ったかを追跡したいとき

### モデル・モード制御

#### `/model [model]`

- **実行内容**: 使用する AI モデルを切り替える。左右キーでエフォートレベルの調整も可能。現在のレスポンスの完了を待たずに即時反映される。
- **効果**: `sonnet`, `opus`, `haiku` などモデルを切り替えることで、速度・精度・コストのバランスを調整できる。
- **ユースケース**:
  - 簡単な質問には Haiku を使い、複雑な設計には Opus を使うなど、タスクに応じてモデルを切り替えたいとき
  - コスト管理のためにモデルを切り替えたいとき

#### `/fast [on|off]`

- **実行内容**: Fast Mode のオン・オフを切り替える。
- **効果**: Fast Mode は同じ Opus 4.6 モデルを約2.5倍速の出力速度で動作させる。モデル自体は変わらない。
- **ユースケース**:
  - 多くのファイルを連続で編集するなど、スループット重視のタスクを行うとき
  - 通常モードで十分な品質が得られるタスクの高速化
  - `Alt+O` のキーボードショートカットでも切り替え可能

#### `/effort [low|medium|high|max|auto]`

- **実行内容**: モデルの思考レベルを設定する。`low`/`medium`/`high` はセッションをまたいで永続化。`max` は現在のセッションのみ（Opus 4.6 限定）。`auto` でモデルデフォルトにリセット。
- **効果**: 低いエフォートは高速だが精度が下がる。高いエフォートは時間がかかるがより深い推論が行われる。
- **ユースケース**:
  - 単純なリネーム作業は `/effort low` で高速に処理
  - 複雑なアーキテクチャ設計は `/effort max` で最大限の思考を割く
  - 引数なしで現在のレベルを確認

#### `/plan [description]`

- **実行内容**: プランモードに直接切り替える。オプションで説明を渡すと、そのタスクについてすぐに計画を開始する。
- **効果**: プランモードではファイルの読み取りのみ可能で、編集や実行は行わない。設計・計画に集中できる。
- **ユースケース**:
  - 実装前にアーキテクチャを検討したいとき（`/plan 認証バグを修正` など）
  - コードレビューのように「読むだけ」で十分なタスク
  - `Shift+Tab` でもモード切替可能

### 情報・ステータス

#### `/help`

- **実行内容**: 利用可能なすべてのコマンド（ビルトイン・カスタム・MCP由来含む）を一覧表示する。
- **ユースケース**:
  - 利用可能なコマンドを確認したいとき
  - カスタムコマンドが正しく登録されているか確認したいとき

#### `/cost`

- **実行内容**: 現在のセッションのトークン使用量とコストを表示する。
- **ユースケース**:
  - API 課金ユーザーがセッション中のコストを監視したいとき
  - 高コストな操作（大量ファイル読み込みなど）の後にコストを確認

#### `/usage`

- **実行内容**: プランの使用量上限とレートリミットの状態を表示する。
- **ユースケース**:
  - サブスクリプションの残り使用量を確認したいとき
  - レートリミットに達しそうかどうかを事前チェック

#### `/doctor`

- **実行内容**: Claude Code のインストール状況と設定を診断する。結果にはステータスアイコンが表示され、`f` キーで問題を自動修正できる。
- **ユースケース**:
  - Claude Code が正常に動作しないとき
  - 新しい環境にセットアップした後の動作確認
  - MCP サーバーやプラグインの接続状態の確認

#### `/status`

- **実行内容**: 設定インターフェースの Status タブを開き、バージョン・モデル・アカウント・接続状態を表示する。Claude の応答中でも即座に実行可能。
- **ユースケース**:
  - 現在接続中のモデルやアカウント情報をすぐに確認したいとき

#### `/stats`

- **実行内容**: 日別の使用量、セッション履歴、連続使用日数（ストリーク）、モデル利用傾向を可視化する。
- **ユースケース**:
  - 自分の Claude Code 利用パターンを振り返りたいとき
  - チームへの利用状況報告資料として

#### `/debug [description]`

- **バンドルスキル**
- **実行内容**: 現在のセッションのデバッグログを有効にし、問題のトラブルシューティングを行う。オプションで問題の説明を渡して分析対象を絞れる。
- **ユースケース**:
  - Claude Code の動作がおかしいとき、内部ログを確認して原因を調査
  - MCP サーバーとの通信問題の診断

#### `/insights`

- **実行内容**: Claude Code のセッション履歴を分析し、プロジェクト領域・インタラクションパターン・フリクションポイントのレポートを生成する。
- **ユースケース**:
  - チームでの Claude Code 活用状況を分析したいとき
  - よくある問題パターンを把握して改善に活かしたいとき

### 設定・権限

#### `/config`

- **エイリアス**: `/settings`
- **実行内容**: タブ付きの設定インターフェースを開き、テーマ・モデル・出力スタイル・その他の設定を変更する。
- **効果**: 対話的に設定を変更でき、変更は即座に反映される。
- **ユースケース**:
  - テーマや出力スタイルを変更したいとき
  - エディタモード（通常/Vim）を切り替えたいとき
  - 各種トグル設定をGUIで一括管理したいとき
- **詳細**: 後述の「`/config` の設定項目詳細」セクションで全項目を解説

#### `/permissions`

- **エイリアス**: `/allowed-tools`
- **実行内容**: ツール権限の allow/ask/deny ルールを管理するインタラクティブダイアログを開く。スコープ別のルール表示、ルールの追加・削除、作業ディレクトリの管理、auto mode の拒否履歴の確認が可能。
- **ユースケース**:
  - `Bash(npm run test *)` のように特定コマンドを自動許可に設定したいとき
  - `.env` ファイルの読み取りを明示的に拒否したいとき
  - auto mode で何が拒否されたかを確認してルールを調整するとき

#### `/theme`

- **実行内容**: カラーテーマを変更する。ライト・ダーク、色覚対応（daltonized）テーマ、ターミナルのカラーパレットを使用する ANSI テーマが選べる。
- **ユースケース**:
  - 周囲の明るさに合わせてライト/ダークを切り替え
  - 色覚特性に合わせたテーマを選択

#### `/sandbox`

- **実行内容**: サンドボックスモードのオン・オフを切り替える。対応プラットフォームのみ。
- **効果**: Bash コマンドがファイルシステムとネットワークから隔離された環境で実行される。
- **ユースケース**:
  - 信頼性が不明なコマンドを安全に試したいとき
  - チームポリシーとしてサンドボックス必須の環境で作業するとき

#### `/hooks`

- **実行内容**: ツールイベントに対するフック設定を表示する。
- **ユースケース**:
  - 設定済みのフック（ツール実行前後のシェルコマンド）を確認したいとき
  - フックが正しく動作しているかデバッグするとき

#### `/color [color|default]`

- **実行内容**: 現在のセッションのプロンプトバーの色を変更する。`red`, `blue`, `green`, `yellow`, `purple`, `orange`, `pink`, `cyan` が利用可能。`default` でリセット。
- **ユースケース**:
  - 複数のターミナルで異なるタスクを進めているとき、視覚的に区別したいとき
  - 本番環境向け作業は赤、開発は緑、のようにリスクレベルを色で表現

### 統合・拡張

#### `/mcp`

- **実行内容**: MCP（Model Context Protocol）サーバーの接続管理と OAuth 認証を行う。
- **ユースケース**:
  - GitHub MCP サーバーなど外部ツール連携を設定・確認したいとき
  - MCP サーバーの OAuth 認証フローを完了させるとき

#### `/install-github-app`

- **実行内容**: Claude GitHub Actions アプリをリポジトリにセットアップする。リポジトリの選択と統合設定のウィザードを案内する。
- **ユースケース**:
  - PR に対して Claude の自動レビュー・自動修正を設定したいとき
  - CI/CD パイプラインに Claude を組み込みたいとき

#### `/install-slack-app`

- **実行内容**: Claude Slack アプリをインストールする。ブラウザで OAuth フローを完了する。
- **ユースケース**:
  - Slack から Claude Code の機能を呼び出したいとき

#### `/plugin`

- **実行内容**: Claude Code プラグインの管理（追加・削除・マーケットプレイス）。
- **ユースケース**:
  - `claude plugin install code-review@claude-plugins-official` のようにプラグインをインストール
  - 利用中のプラグインの一覧確認や削除

#### `/reload-plugins`

- **実行内容**: アクティブなプラグインをすべてリロードして変更を適用する。再起動不要。
- **ユースケース**:
  - プラグインの設定を変更した後、セッションを中断せずに反映させたいとき

#### `/skills`

- **実行内容**: 利用可能なスキル一覧を表示する。
- **ユースケース**:
  - 現在有効なスキル（バンドル・カスタム・プラグイン由来）を確認したいとき

#### `/agents`

- **実行内容**: サブエージェント設定を管理する。
- **ユースケース**:
  - カスタムサブエージェントの設定確認・追加・変更

#### `/ide`

- **実行内容**: IDE 統合の管理とステータス表示。
- **ユースケース**:
  - VS Code や JetBrains との連携状態を確認・管理するとき

#### `/remote-control`

- **エイリアス**: `/rc`
- **実行内容**: 現在のセッションを claude.ai からリモートコントロール可能にする。
- **ユースケース**:
  - ターミナルで起動した Claude Code をブラウザの claude.ai から操作したいとき
  - モバイルから進行状況を確認・操作したいとき

#### `/teleport`

- **エイリアス**: `/tp`
- **実行内容**: Web上のClaude Codeセッションをローカルターミナルに引き込む。ピッカーが開き、ブランチと会話を取得する。
- **ユースケース**:
  - claude.ai 上で開始したセッションをローカル環境で引き継ぎたいとき
  - Web で計画した作業をターミナルで本格的に実行したいとき

#### `/desktop`

- **エイリアス**: `/app`
- **実行内容**: 現在のセッションを Claude Code Desktop アプリで継続する。macOS/Windows のみ。
- **ユースケース**:
  - ターミナルからデスクトップアプリに切り替えてGUIで作業を続けたいとき

#### `/autofix-pr [prompt]`

- **実行内容**: 現在のブランチの PR を監視し、CI 失敗やレビューコメントに対して自動修正を push する Claude Code on the web セッションを起動する。
- **ユースケース**:
  - PR を作成した後、CI の修正やレビュー対応を自動化したいとき
  - `/autofix-pr lint とタイプエラーだけ修正して` のように対象を絞ることも可能

#### `/web-setup`

- **実行内容**: ローカルの `gh` CLI 認証情報を使って GitHub アカウントを Claude Code on the web に接続する。
- **ユースケース**:
  - `/schedule` や web セッション機能を初めて使うとき

#### `/remote-env`

- **実行内容**: `--remote` で開始する web セッションのデフォルトリモート環境を設定する。
- **ユースケース**:
  - Web セッションで使用するコンテナ環境やリソースを事前設定しておきたいとき

### ワークフロー（バンドルスキル）

以下のコマンドは**バンドルスキル**であり、Claude にプロンプトとして渡される。自動的にコンテキストに応じて Claude 自身が呼び出す場合もある。

#### `/batch <instruction>`

- **実行内容**: コードベース全体に対する大規模な変更を並列で実行する。コードベースを調査し、作業を5〜30の独立したユニットに分解して計画を提示する。承認後、各ユニットごとに独立した git worktree で背景エージェントを起動し、実装・テスト・PR作成まで行う。
- **ユースケース**:
  - `src/` 配下のコンポーネントを Solid から React にマイグレーション
  - 全ファイルの import パスを一括変更
  - 大規模なリネーム・リファクタリング

#### `/simplify [focus]`

- **実行内容**: 最近変更したファイルについて、コードの再利用性・品質・効率性を3つのレビューエージェントが並列でレビューし、問題を修正する。オプションで焦点を指定可能。
- **ユースケース**:
  - 実装後に「もう少し改善できないか」をチェックしたいとき
  - `/simplify メモリ効率に注目` のように特定の観点に絞ったレビュー

#### `/loop [interval] [prompt]`

- **エイリアス**: `/proactive`
- **実行内容**: プロンプトを繰り返し実行する。インターバルを省略すると Claude が自動でペーシング。プロンプトも省略すると自律的なメンテナンスチェック（または `.claude/loop.md` のプロンプト）を実行する。
- **ユースケース**:
  - `/loop 5m デプロイが完了したか確認` のように定期的なポーリング
  - `/loop` だけで自律的にコードベースの問題を探して修正するプロアクティブ運用

#### `/claude-api`

- **実行内容**: プロジェクトの言語に合わせた Claude API リファレンス（Python, TypeScript, Java, Go, Ruby, C#, PHP, cURL）と Managed Agents リファレンスを読み込む。tool use, streaming, batches, structured outputs, 一般的な落とし穴をカバー。`anthropic` や `@anthropic-ai/sdk` の import を検出して自動起動もする。
- **ユースケース**:
  - Claude API を使ったアプリケーションを開発するとき
  - Anthropic SDK の使い方を確認しながらコーディングするとき

#### `/security-review`

- **実行内容**: 現在のブランチの未コミット変更に対してセキュリティ脆弱性の分析を行う。git diff をレビューし、インジェクション・認証問題・データ漏洩などのリスクを特定する。
- **ユースケース**:
  - PR を出す前にセキュリティ上の問題がないか最終チェック
  - 外部入力を扱うコードの変更時に脆弱性がないか確認

### サイドクエスチョン

#### `/btw <question>`

- **実行内容**: 会話履歴に追加されない一時的なサイドクエスチョンを投げる。Claude の応答中でも実行可能。ツールアクセスはなく、現在のコンテキスト内の情報のみで回答する。
- **効果**: メインの会話を汚さずに素早く質問でき、応答はオーバーレイで表示される。
- **ユースケース**:
  - 長い作業の途中で「あの設定ファイルの名前なんだっけ？」と確認
  - メインタスクを中断せずに関連する疑問を解決
  - `Space`/`Enter`/`Escape` で回答を閉じて作業に戻る

## `/config` の設定項目詳細

`/config` コマンドで開く設定インターフェースは、`settings.json` および `~/.claude.json` に保存される設定を対話的に操作する。以下に主要な設定項目を整理する。

### インタラクティブUIで変更できる主要設定

| 設定項目 | 説明 | 保存先 |
|---------|------|--------|
| **Theme** | シンタックスハイライトのカラーテーマ | `~/.claude.json` |
| **Model** | デフォルトで使用するモデル | `settings.json` |
| **Output style** | 出力スタイル（簡潔/詳細など） | `settings.json` |
| **Editor mode** | Normal/Vim の切り替え | `~/.claude.json` |
| **Extended thinking** | 拡張思考モード（thinking mode）の有効/無効 | `settings.json` |
| **Show thinking summaries** | 拡張思考の要約をインタラクティブセッションで表示するか | `settings.json` |
| **Show turn duration** | ターンの所要時間表示（例: "Cooked for 1m 6s"） | `~/.claude.json` |
| **Terminal progress bar** | ターミナルのプログレスバー表示 | `~/.claude.json` |
| **Prompt suggestions** | プロンプト入力時のサジェスチョン表示 | `settings.json` |
| **Spinner tips** | 処理中のスピナーにTipsを表示するか | `settings.json` |
| **Auto-connect to IDE** | 外部ターミナルから IDE に自動接続 | `~/.claude.json` |
| **Auto-install IDE extension** | VS Code ターミナルから IDE 拡張を自動インストール | `~/.claude.json` |

### `settings.json` の主要キー

`settings.json` はユーザースコープ（`~/.claude/settings.json`）、プロジェクトスコープ（`.claude/settings.json`）、ローカルスコープ（`.claude/settings.local.json`）の3階層で設定できる[[2]](#参考リンク)。

#### 基本設定

| キー | 説明 | 例 |
|-----|------|-----|
| `model` | デフォルトモデルの指定 | `"claude-sonnet-4-6"` |
| `language` | Claude の応答言語 | `"japanese"` |
| `outputStyle` | 出力スタイル | `"Explanatory"` |
| `effortLevel` | エフォートレベルの永続化 | `"medium"` |
| `availableModels` | 使用可能なモデルの制限 | `["sonnet", "haiku"]` |

#### 権限設定

| キー | 説明 | 例 |
|-----|------|-----|
| `permissions.allow` | 自動許可するツール操作 | `["Bash(npm run test *)"]` |
| `permissions.ask` | 確認を求めるツール操作 | `["Bash(git push *)"]` |
| `permissions.deny` | 拒否するツール操作 | `["Read(./.env)"]` |
| `defaultMode` | デフォルトの権限モード | `"acceptEdits"` |

#### フック・自動化

| キー | 説明 | 例 |
|-----|------|-----|
| `hooks` | ライフサイクルイベントのフック | [hooks ドキュメント](https://code.claude.com/docs/en/hooks)参照 |
| `env` | セッションに適用する環境変数 | `{"FOO": "bar"}` |

#### サンドボックス設定

| キー | 説明 | 例 |
|-----|------|-----|
| `sandbox.enabled` | Bash サンドボックスの有効化 | `true` |
| `sandbox.autoAllowBashIfSandboxed` | サンドボックス内のBashを自動許可 | `true` |
| `sandbox.filesystem.allowWrite` | サンドボックス内の書き込み許可パス | `["/tmp/build"]` |
| `sandbox.network.allowedDomains` | 許可するドメイン | `["github.com", "*.npmjs.org"]` |

#### 表示・UI カスタマイズ

| キー | 説明 | 例 |
|-----|------|-----|
| `attribution` | コミット・PRの帰属表記カスタマイズ | `{"commit": "AI生成", "pr": ""}` |
| `spinnerVerbs` | スピナーの動詞カスタマイズ | `{"mode": "append", "verbs": ["考え中"]}` |
| `spinnerTipsOverride` | スピナーTipsの上書き | `{"excludeDefault": true, "tips": ["独自Tips"]}` |
| `prefersReducedMotion` | アニメーション軽減 | `true` |
| `statusLine` | カスタムステータスライン | [statusLine ドキュメント](https://code.claude.com/docs/en/statusline)参照 |

#### エンタープライズ・管理者向け設定

| キー | 説明 |
|-----|------|
| `companyAnnouncements` | 起動時のアナウンスメッセージ |
| `forceLoginMethod` | ログイン方法の制限（`claudeai` / `console`） |
| `forceLoginOrgUUID` | 特定組織のログインを強制 |
| `disableAutoMode` | auto mode の無効化 |
| `allowManagedPermissionRulesOnly` | 管理者ルールのみ適用 |
| `allowManagedMcpServersOnly` | 管理者指定の MCP サーバーのみ許可 |

### 設定スコープの優先順位

設定は以下の優先順位で適用される（上が最優先）[[2]](#参考リンク):

1. **Managed**（最高） - IT部門によるポリシー。上書き不可
2. **コマンドライン引数** - セッション限定の一時的上書き
3. **Local**（`.claude/settings.local.json`） - 個人のプロジェクト固有設定
4. **Project**（`.claude/settings.json`） - チーム共有のプロジェクト設定
5. **User**（`~/.claude/settings.json`） - 個人のグローバル設定

## まとめ

Claude Code のスラッシュコマンドは約60種類あり、セッション管理からセキュリティレビュー、大規模リファクタリングまで幅広いワークフローをカバーしている。特に以下のポイントを押さえておくと活用の幅が広がる:

- **日常的に使うコマンド**: `/compact`, `/model`, `/fast`, `/cost`, `/config`
- **探索・計画**: `/plan`, `/context`, `/btw`, `/diff`
- **チーム・CI連携**: `/install-github-app`, `/autofix-pr`, `/batch`, `/schedule`
- **品質向上**: `/simplify`, `/security-review`
- **設定の階層**: ユーザー → プロジェクト → ローカルの3層 + 管理者による強制設定

`/help` を実行すれば、現在の環境で利用可能なコマンドをいつでも確認できる。カスタムスキルを `.claude/commands/` に追加すれば、プロジェクト固有のワークフローもスラッシュコマンドとして統合できる。
